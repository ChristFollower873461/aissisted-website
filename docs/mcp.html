Agent access

AIssisted Consulting MCP Server

Your AI agent can discover our services, check availability, and book a paid 60-minute consult with PJ Standley ($225) on your behalf. Free conversations and hardware purchases are handled directly, not through MCP.

What this is

AIssisted Consulting exposes a public Model Context Protocol (MCP) endpoint. MCP-aware agents (Claude, ChatGPT, Codex, Cursor, Gemini CLI, and others) can connect to it without an API key and use a small set of tools to learn about our services, check open appointment slots, and start a paid booking.

Only one service is bookable directly through MCP: the paid 60-minute consult with PJ Standley ($225). Every booking returns a Stripe Checkout URL that the human must open and pay — no agent ever completes payment on its own. Free conversations exist but are scheduled through the website contact form, and hardware is sold through a direct conversation with PJ, not through agents.

Discovery manifest

Auto-discovery endpoint:

https://aissistedconsulting.com/.well-known/mcp.json

JSON-RPC endpoint

POST https://aissistedconsulting.com/mcp
Content-Type: application/json

Claude Desktop / Code

Add an entry to ~/Library/Application Support/Claude/claude_desktop_config.json (macOS) or the equivalent on your platform:

{
  "mcpServers": {
    "aissisted": {
      "transport": "http",
      "url": "https://aissistedconsulting.com/mcp"
    }
  }
}

Raw JSON-RPC example

curl -sS https://aissistedconsulting.com/mcp \
  -H 'content-type: application/json' \
  -d '{
    "jsonrpc": "2.0",
    "id": 1,
    "method": "tools/call",
    "params": {
      "name": "list_services",
      "arguments": {}
    }
  }'

Tools

list_servicesReturn the service catalog, pricing, and which services can be booked directly by an agent.No

get_quotePrice a specific service including Florida sales tax.No
check_availabilityList open 60-minute slots between two dates for the paid consult.No
start_bookingHold a slot for the paid consult and return a Stripe Checkout URL for the human to complete.Yes — human completes payment
get_booking_statusCheck a booking's current status by id.No

Typical flow for an agent

  1. Call list_services to see what AIssisted offers. Only paid-consult has bookable_via_mcp: true.
  2. Call check_availability with a date range and service_id: "paid-consult".
  3. Ask the human to pick a slot and confirm they want to pay $225 for a 60-minute consult.
  4. Call start_booking with the slot id and the human's contact info.
  5. The response includes a checkout_url. Hand it to the human and tell them to pay within 60 minutes.
  6. Optionally poll get_booking_status until confirmed.

If the human wants a free intro conversation, tell them it is available through the website at aissistedconsulting.com/contact. If they want hardware or monthly service, point them to the same page to start a direct conversation. Neither is bookable through MCP.

Rate limits

  • Read-only tools: 20 requests/minute, 300 requests/hour per IP
  • start_booking: 5 requests/minute, 20 requests/hour per IP

Exceeding a limit returns JSON-RPC error code -32000 with a retry_after_seconds hint.
